# Budil Playbooks

## 6月なのに予約が少ない時

トリガー：

- 7日以内の予約が0〜2件
- エアコン繁忙期なのに問い合わせが少ない
- 広告を出しているのに反応が弱い

優先行動：

1. Google広告の表示・クリック・CTA確認
2. LPのLINE/予約CTA確認
3. 既存客へのLINE追客
4. 法人営業リストを10件確認
5. 洗濯機クリーニング投稿
6. 完全分解LPの訴求確認

## 空き日が3日以上ある時

優先行動：

1. 今日営業すべき会社を確認
2. 興味あり / 見積り中 / 日程調整中へ再連絡
3. 既存顧客へ季節提案
4. Googleビジネスプロフィール投稿
5. SNS投稿
6. 法人向け営業文面を作成

## Google広告で表示はあるがクリックが少ない時

見る場所：

- キーワード
- 広告文
- 地域
- 検索語句
- LPとの一致

次の一手：

- 広告文を見直す
- 地域名を強める
- 料金訴求を入れる
- 完全分解と通常洗浄を分けて考える

## Google広告でクリックはあるがCTAが少ない時

見る場所：

- LPファーストビュー
- 料金到達率
- CTA位置
- LINEボタン
- 電話ボタン
- 不安解消要素

次の一手：

- CTAを上に追加
- 料金の見せ方を確認
- 写真・口コミ・実績を強化
- スマホ表示を確認

## CTAはあるが問い合わせが少ない時

見る場所：

- LINE導線
- 予約導線
- 問い合わせ文面
- 価格への不安
- 対応エリア
- 作業中は電話に出られない旨の説明

次の一手：

- LINE相談を強める
- 写真を送ると早いと説明
- 予約前の不安をFAQで消す
- 電話よりLINEが確実な導線にする

## 問い合わせはあるが成約しない時

見る場所：

- 価格説明
- 完全分解と通常の違い
- 日程調整
- 遠方料金
- 追加料金説明
- 返信速度

次の一手：

- 見積りテンプレ改善
- 料金表を分かりやすくする
- 施工写真を添付
- 日程候補を複数出す

## 雨・湿気・カビが強い時

優先サービス：

- 洗濯機クリーニング
- 浴室
- レンジフード
- エアコン内部カビ
- 完全分解

投稿テーマ：

- ニオイ
- カビ
- 湿気
- 子どもがいる家庭の安心
- 洗濯物の生乾き臭

## 台風前後

台風前：

- 無理な予約を入れない
- 家族優先
- 外作業・移動リスクを確認

台風後：

- エアコン不具合
- 水漏れ
- 室外機まわり
- 湿気・カビ訴求
- 予約再調整

## 予約が埋まっている時

優先行動：

- 高単価メニューを優先
- 完全分解を優先
- 遠方低単価は慎重に判断
- 既存客を大事にする
- 無理に広告費を増やさない

## 法人営業を強める時

対象：

- 不動産
- 管理会社
- 民泊
- 美容室
- 整骨院
- 飲食店
- 学童
- 小規模施設
- 事務所

次の一手：

- 営業プリセット選択
- 営業文面生成
- 初回連絡
- 次アクション日設定
- 興味ありなら再連絡
- 見積り中なら日程提案

## 現場後の追客

作業後にやること：

- お礼LINE
- 口コミ依頼
- 次回提案
- 洗濯機やレンジフード提案
- 法人なら定期清掃提案
- Budilに最終連絡日を残す

## アナリティクス外部チェック（GA4 / Search Console → Budil保存）

### 目的

GA4 / Search Console の画面上の数値を読み取り、Budil外部チェックへ保存し、今日やること候補へ変換する。

売上確定ではない。Browser Automation / 手動ログイン / 画面読み取りによる半自動運用を前提とする。Google API / OAuth / バックエンド連携は将来検討。

### 実行トリガー

- 週1回
- LP改善前後
- 広告を止めた/変えた後
- 検索流入が気になる時
- 売上が落ちた/増えた理由を見たい時

### 期間ルール

原則：**直近28日間**

GA4で28日間切り替えが難しい場合は、**画面上の表示期間**として記録し、`## 注意・未確認` または `## 注意・ノイズ候補` に残す。

GA4 と Search Console で期間が一致しないことがある。横並び比較は参考程度。

### 役割分担

| 役割 | ツール |
|------|--------|
| 外部サービスを見る目 | Browser番頭 |
| その場の読み取り・検査員 | Cursor Browser Automation |
| 記録・判断・行動候補化 | Budil（集客・需要 → 外部チェック） |

### 読み取り対象

**GA4**

- アクティブユーザー
- 新規ユーザー
- セッション
- 表示回数
- イベント数
- キーイベント
- 流入元
- ページ/LP上位
- 目立つ変化
- 海外アクセスなどノイズ

**Search Console**

- 合計クリック数
- 合計表示回数
- 平均CTR
- 平均掲載順位
- 上位クエリ
- 上位ページ
- 表示多いがクリック0のクエリ
- 順位が良いのにクリックが少ないクエリ

### 手順（半自動・2026-06-25 検証済み）

1. Cursor Browser Automation で GA4 を開く（ログイン済みタブ）
2. Search Console の検索パフォーマンスを開く（可能なら28日間）
3. 画面上の数値のみ読み取る（推測しない）
4. `【Budil貼り付け用】` 形式に変換する
5. Budil 公開 URL（https://teruya1229.github.io/budil/）→ 集客・需要 → 外部チェックへ貼り付ける
6. 保存する（保存IDが画面に表示される）
7. 最新レポート・履歴・経営ホームカードを確認する
8. 必要なら「行動候補に追加」または「今日やることに追加」

### Budil貼り付け用テンプレート

```md
【Budil貼り付け用】
種別：外部チェックレポート
確認日：YYYY-MM-DD
確認対象：GA4 / Search Console
対象期間：GA4：直近28日間または画面上の表示期間 / Search Console：直近28日間または画面上の表示期間

## 予定候補
- 未確認

## 需要候補
- Search Consoleから分かる検索需要を書く
- 表示が多いクエリ、クリック0のクエリ、順位が良いクエリを優先する

## アナリティクス候補
- GA4から分かるアクセス・流入・イベント変化を書く
- Organic Search / Paid Search / Direct の変化を書く
- ページ別の反応を書く

## GBP反応
- 未確認

## 広告異常
- Google広告管理画面は未確認
- GA4上のPaid Search変化があれば注意として書く
- 売上確定ではない

## 注意・ノイズ候補
- 海外トラフィック急増など実需とズレる可能性を書く
- 期間不一致を書く
- 画面上の数値のみであることを書く

## 今日やること候補
- 数値から次にやるべき改善候補を3〜6件書く
- LP改善、タイトル改善、記事追加、ノイズ確認など具体的に書く

## 注意・未確認
- 正確に取得できなかった期間や項目を書く
- 画面上の数値のみ使用したことを書く
- 売上確定ではないことを書く
```

### 禁止事項

- GA4設定変更
- Search Console設定変更
- Google広告設定変更
- 推測で数値を作る
- 売上確定扱いする
- 未確認項目を確定表現にする
