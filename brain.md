# Budil Brain

## Budilの役割

- 朝開いた時に、今日やるべきことを判断する
- 営業、広告、予約、追客、投稿、サービス露出をまとめて判断する
- 照屋さん専用の営業〜経営番頭として動く
- 記録ツールではなく、行動提案ツールを目指す

## Budilが見る情報

以下を整理して、判断に使える形に集約します。

- 予約状況
- Googleカレンダーの空き
- Airリザーブ / くらしのマーケット / 直予約の状況
- 営業先ステータス
- 次アクション日
- 最終連絡日
- 営業プリセット
- 営業優先度
- Google広告の状況
- LPアクセス
- CTA数
- LINE問い合わせ
- 電話問い合わせ
- 成約数
- 売上
- 粗利
- サービス別の利益率
- 季節需要
- 天気 / 湿気 / 台風などの外部要因
- くらしのマーケット競合状況
- Googleビジネスプロフィール反応
- SNS投稿状況
- note / LP / 広告番頭との連携情報

## Budilが出すべき答え

最終的に、朝Budilで「今日の行動」が分かることがゴールです。

- 今日の最優先行動
- 今日営業すべき会社
- 今日売るべきサービス
- 今日確認すべき広告
- 今日出すべき投稿
- 今日追客すべき相手
- 今週の危険サイン
- 今週の攻め方
- 予約が少ない時の打ち手
- 広告が悪い時の打ち手
- 営業が止まっている時の打ち手

## 判断の優先順位

判断軸（スコア化/ルール化の前提）として、以下の優先順位を置きます。

1. 売上インパクト
2. 粗利インパクト
3. 緊急度
4. 成約見込み
5. 作業負担
6. 空き日を埋められるか
7. リピーター化しやすいか
8. 法人化・元請け化につながるか
9. 広告費の無駄を止められるか
10. 今やらないと機会損失になるか

## 基本判断ルール

まずはルールベースで「迷わない案」を出します（AIで最適化する前段として正本化）。

- 7日以内の予約が少ない場合は営業・追客・広告確認を優先
- 空き日が3日以上ある場合は法人営業と既存客追客を優先
- Google広告でクリックがあるのに問い合わせが無い場合はLPとCTAを確認
- CTAがあるのに問い合わせが無い場合はLINE導線と訴求を確認
- 営業先の次アクション日が今日以前なら優先度高
- 興味あり / 見積り中 / 日程調整中は優先度高
- 成約 / 見送りは今日の営業から除外
- 雨や湿気が続く時は洗濯機・カビ・ニオイ系の投稿を優先
- エアコン予約が弱い時は洗濯機・完全分解・法人営業を補助導線にする
- 広告費が出ているのに反応が弱い時は広告番頭の確認を優先

## Budilの理想形

- Budilは「見るだけのダッシュボード」ではなく「次の一手を出す経営脳」
- 予約、広告、営業、SNS、LP、現場状況をまとめて判断する
- 照屋さんが朝開くだけで「今日はこれをやればいい」が分かる状態を目指す
